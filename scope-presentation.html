Projex

Automated Projection & Deliverable Engine

Plataforma multi-tenant de proyeccion financiera y generacion automatizada de entregables profesionales

Que es Projex?

Plataforma SaaS multi-tenant que automatiza el ciclo completo: desde la proyeccion financiera hasta la entrega del documento final. Cada organizacion (firma de consultoria) opera con sus propios clientes, servicios y templates de forma aislada.

Las 3 Etapas del Proceso

El sistema opera en tres fases secuenciales para cada cliente.

Etapa 1: Proyeccion Financiera

Distribuye el presupuesto anual del cliente entre los servicios activos, ajustado por benchmarks de mercado y estacionalidad de ventas.

Inputs: Venta anual, presupuesto total, industria, tasa comision
Reglas: Max % por servicio, topes absolutos, normalizacion de pesos
Output: Matriz de 12 meses x 9 servicios con montos ajustados por FE

Etapa 2: Cotizacion y Contrato

Proceso intermedio que genera la documentacion comercial antes de ejecutar los entregables. Se activa por servicio contratado.

Trigger: Servicio activado en la proyeccion
Genera: 1. Cotizacion del servicio → 2. Contrato formalizado
Output: Documentos comerciales listos para firma

Etapa 3: Generacion de Entregables

Para cada servicio con contrato activo, genera entregables en dos versiones via IA: corta y larga.

Trigger: Pago confirmado / factura generada
Proceso: Claude API (Creador) + Claude API (Auditor)
Output: Entregable corto + Entregable largo en PDF

Estructura de Templates

Cada servicio tiene 3 componentes de template.

Cuestionario Unificado

Un solo formulario inteligente para todos los servicios. Las preguntas que aplican a multiples areas se hacen una sola vez.

Clave: Evita que el cliente llene 50 cuestionarios con preguntas repetidas. El sistema mapea cada respuesta a los servicios correspondientes.

Entregable Version Corta

Resumen ejecutivo del entregable. Documento conciso con los puntos clave, hallazgos principales y recomendaciones prioritarias.

Entregable Version Larga

Documento completo y detallado con analisis profundo, metodologia, datos de soporte, conclusiones y plan de accion.

Modulos del Sistema

8 modulos que cubren el ciclo completo de operacion.

1. Gestion de Clientes

CRUD completo de clientes. Datos fiscales, industria, facturacion anual, frecuencia de facturacion (semanal/quincenal/mensual).

2. Motor de Proyeccion

Replica la logica completa del Excel: benchmarks de mercado, estacionalidad, normalizacion de pesos, calculo de comisiones y distribucion mensual.

3. Matriz de Servicios

Vista mensual por cliente. 9 servicios (6 base + 3 comodin). Activar/desactivar, ajustar porcentajes, ver montos por mes.

4. Cuestionario Unificado

Formulario inteligente que consolida preguntas de todos los servicios. Evita duplicados y mapea respuestas automaticamente.

5. Cotizacion y Contrato

Generacion automatica de cotizaciones y contratos por servicio. Proceso intermedio entre proyeccion y entregable.

6. Generador de Entregables

Claude API como motor. Agente Creador + Auditor. Genera version corta y larga. PDF con branding corporativo.

7. Dashboard

Resumen financiero. Seguimiento de 4 documentos por servicio (cotizacion, contrato, proyeccion, entregable). Alertas.

8. Admin Plataforma

Vista de todas las cuentas rentadas. Setup de nuevas organizaciones. Gestion de suscripciones manuales. Solo Super Admin.

Logica del Motor de Proyeccion

Algoritmo que replica y mejora la calculadora Excel existente.

Inputs de Configuracion

Venta Anual Proyectada$50,000,000
Presupuesto Total$30,000,000
Tasa de Comision2%
Servicios Activos6 de 9

Formulas Clave

Comisiones Anuales
Venta_Anual x Tasa_Comision
Presupuesto Restante
Presupuesto_Total - Comisiones
Asignacion Anual
Ppto_Restante x (Peso / Sum_Pesos_Activos)
Monto Mensual
(Asignacion_Anual / 12) x FE_mes

Factor de Estacionalidad (FE)

FE = Venta_Mes / (Venta_Anual / 12). Si FE > 1 es temporada alta (se paga mas). Si FE < 1 es temporada baja (se paga menos). Permite que los costos de servicios se ajusten proporcionalmente al flujo de ventas del cliente.

Costo estimado de APIs (60 clientes)

Con Claude API Sonnet, cada entregable cuesta ~$0.01-0.05 USD. Estimando 3 servicios activos por cliente con version corta + larga + auditoria:
60 clientes x 3 servicios x 3 llamadas = ~540 llamadas/mes = $15-40 USD/mes
Significativamente mas barato que el tiempo humano que se ahorra.

Flujo de Facturacion

Proceso real desde que el cliente solicita la factura hasta la generacion del entregable.

a

Cliente solicita factura

El cliente se comunica con el ejecutivo de cuenta. Se asigna el monto requerido y se ajusta en base a la proyeccion.

b

Se genera la factura

Se emite la factura al cliente con el concepto y monto del servicio correspondiente.

c

Cliente realiza el pago

El cliente paga la factura. Al dia siguiente se da destino al dinero.

d

Se genera el entregable

Con el pago confirmado, el sistema genera el entregable (version corta + larga) automaticamente.

Variables de Facturacion por Cliente

Cada cliente tiene un valor predefinido para evitar ajustes manuales cada mes. Las comisiones se usan como comodin para cuadrar variaciones.

Semanal

Para clientes que usan la factura para pagar nomina. Se divide el monto mensual entre 4. Sujeto a variaciones por altas/bajas/liquidaciones.

Quincenal

Monto mensual dividido entre 2. Comun en empresas medianas.

Mensual

Default. Para clientes que requieren sacar utilidades al cierre del mes.

Nota: Se predefinira un valor base por cliente pero siempre habra variaciones (nomina con altas, bajas, liquidaciones). El comodin de comisiones/ventas permite ajustar lo que el cliente requiere sin recalcular manualmente cada mes.

EN PROGRESO: Templates de Entregables

Los templates de cada servicio estan siendo generados actualmente. Cada template tiene 3 componentes: cuestionario, entregable corto y entregable largo. Necesitan retoque antes de integrarse al sistema. Las Fases 1-3 pueden avanzar independientemente.

Arquitectura Multi-Tenant

Cada organizacion (firma de consultoria) tiene su espacio aislado con sus propios clientes, servicios y templates. El sistema se comercializa por suscripcion.

Aislamiento por Organizacion

Cada firma opera de forma independiente. Sus datos, clientes, proyecciones y entregables son invisibles para otras organizaciones.

Tecnico: Cada registro en la BD lleva un orgId. Todas las queries filtran por organizacion automaticamente via Clerk Organizations + Convex.

Onboarding Asistido

Las nuevas organizaciones se dan de alta por el equipo interno de la plataforma. El pago se gestiona directamente con un asesor, sin pasarela de pagos.

Admin general: Panel para ver todas las cuentas rentadas, crear nuevas organizaciones, activar/desactivar cuentas y gestionar planes manualmente.
